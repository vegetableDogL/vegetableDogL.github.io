1、Spring02

1、常用注解

1、启动注解功能（开启组件扫描）

  • 启动注解扫描，加载类中配置的注解项，在applicationContext.xml配置

    <context:component-scan base-package="packageName"/>

  • 说明：

    • 在进行包所扫描时，会对配置的包及其子包中所有文件进行扫描

    • 扫描过程是以文件夹递归迭代的形式进行的

    • 扫描过程仅读取合法的java文件

    • 扫描时仅读取spring可识别的注解

    • 扫描结束后会将可识别的有效注解转化为spring对应的资源加入IoC容器

  • 注意：

    • 无论是注解格式还是XML配置格式，最终都是将资源加载到IoC容器中，差别仅仅是数据读取方式不同
    • 从加载效率上来说注解优于XML配置文件

2、bean的定义

@Component @Controller @Service @Repository

类注解

作用：设置该类为spring管理的bean

  • 名称：@Component @Controller @Service @Repository

  • 类型：类注解

  • 位置：类定义上方

  • 作用：设置该类为spring管理的bean

  • 范例：

    @Component/@Component("id值")
public class ClassName{}

  • 说明：

    • @Controller、@Service 、@Repository是@Component的衍生注解，功能同@Component

  • 相关属性

    • value（默认）：定义bean的访问id

3、bean的作用域

  • 名称：@Scope

  • 类型：类注解

  • 位置：类定义上方

  • 作用：设置该类作为bean对应的scope属性

  • 范例：

    @Scope/@Scope("prototype")        //默认值是singleton
public class ClassName{}

  • 相关属性

    • value（默认）：定义bean的作用域，默认为singleton

  • 名称：@PostConstruct、@PreDestroy

  • 类型：方法注解

  • 位置：方法定义上方

  • 作用：设置该类作为bean对应的生命周期方法

  • 范例：

代码演示：

/*
    1. bean标签配置

            相当于配置了id -> Class

        2). 四个注解,效果完全相同
            @Component    @Controller     @Service          @Repository
             组件           控制器(web)     业务层(service)   仓库(dao)

    2. 作用范围的配置

    }

    public void init(){
        System.out.println("init");
    }
    @PreDestroy
    public void destroy(){
        System.out.println("destroy");
    }

5、bean的非引用类型属性注入

  • 名称：@Value

  • 类型：属性注解、方法注解

  • 位置：属性定义上方，方法定义上方

  • 作用：设置对应属性的值或对方法进行传参

  • 范例：

    @Value("${jdbc.username}")
private String username;

  • 说明：

    • value值仅支持非引用类型数据，赋值时对方法的所有参数全部赋值

    • value值支持读取properties文件中的属性值，需要在类上配置@PropertySource(“ classpath:data.properties “),或者在xml配置外部文件扫描<context:property-placeholder location=”classpath:jdbc.properties”/>

    • value值支持SpEL

    • @value注解如果添加在属性上方，可以省略set方法（反射属性注入值，和xml配置的set注入不同）

  • 相关属性

    • value（默认）：定义对应的属性值或参数值

6、bean的引用类型属性注入

  • 名称：@Autowired、@Qualifier

  • 类型：属性注解、方法注解

  • 位置：属性定义上方，方法定义上方

  • 作用：设置对应属性的对象或对方法进行引用类型传参

  • 范例：

    @Autowired(required = true)
@Qualifier("userDao")
private UserDao userDao;

  • 说明：

    1. @Autowired
     原理:
         注入的引用类型对象(来自IOC容器)
         如果IOC容器中只有一个此类型的对象,直接注入
         如果IOC容器中有多个此类型的对象,按照IOC容器中对象的名称注入,如果名称不匹配报错
     属性: 了解,一般不写
         required: 
             true: 此对象必须注入成功,若不成功则报错. 默认值
             false: 可以注入不成功,此对象为null
2. @Qualifier
        1). 作用:在自动按照类型注入的基础之上,再按照bean的id注入.
        2). 注意:此注解不能独立使用,它必须和Autowired一起使用

代码演示：

//定义bean，后面添加bean的id
@Component("userService")
public class UserServiceImpl implements UserService {

    @Value("zs")
    String name;
    @Value("18")
    int age;
    /* 
    1. @Autowired
         原理:
             注入的引用类型对象(来自IOC容器)
             如果IOC容器中只有一个此类型的对象,直接注入
             如果IOC容器中有多个此类型的对象,按照IOC容器中对象的名称注入,如果名称不匹配报错
         属性: 了解,一般不写
             required: 
                 true: 此对象必须注入成功,若不成功则报错. 默认值
                 false: 可以注入不成功,此对象为null
    2. @Qualifier
            1). 作用:在自动按照类型注入的基础之上,再按照bean的id注入.
            2). 注意:此注解不能独立使用,它必须和Autowired一起使用   
        */
    @Autowired(required = true)
    @Qualifier("userDao2")
    UserDao userDao;


    public void setName(String name) {
        this.name = name;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public void setUserDao(UserDao userDao) {
        this.userDao = userDao;
    }

    @Override
    public void save() {
        System.out.println("UserServiceImpl run..." + name + "," + age);
        userDao.save();
    }

}

7、加载properties文件

  • 名称：@PropertySource

  • 类型：类注解

  • 位置：类定义上方

  • 作用：加载properties文件中的属性值

  • 范例：

    @PropertySource(value = "classpath:filename.properties")
public class ClassName {
    @Value("${propertiesAttributeName}")
    private String attributeName;
}

  • 相关属性

    • value（默认）：设置加载的properties文件名

代码演示：

@Repository("bookDao")
//加载properties文件中的属性值
@PropertySource("classpath:data.properties")
public class BookDaoImpl implements BookDao {

    @Value("${name}")
    String name;
    @Value("${password}")
    String password;

    public void setName(String name) {
        this.name = name;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    @Override
    public void save() {
        System.out.println("BookDaoImpl run..." + name + "," + password);
    }
}

8、加载第三方资源

  • 名称：@Bean

  • 类型：方法注解

  • 位置：方法定义上方

  • 作用：设置该方法的返回值作为spring管理的bean

  • 范例：

    @Bean("dataSource")
public DruidDataSource createDataSource() {    return ……;    }

  • 说明：

    • 因为第三方bean无法在其源码上进行修改，使用@Bean解决第三方bean的引入问题
    • 该注解用于替代XML配置中的静态工厂与实例工厂创建bean，不区分方法是否为静态或非静态
    • @Bean所在的类必须被spring扫描加载，否则该注解无法生效

  • 相关属性

    • value（默认）：定义bean的访问id

代码演示：

@Component        //必须添加，否则spring不会识别这个类，则下面的@Bean也用不了
public class MyClassFactory {
    /**
     * @Bean:
     *      将方法的返回值存放到IOC容器中
     *      ioc容器中key的名称默认为 方法的名称
     *      value属性: 设置存放到ioc容器时bean的名称
     */
    @Bean("fb")
    public MyClass getMyClass(){
        MyClass mc = new MyClass();
        mc.setName("lisi");
        mc.setAge(18);
        return mc;
    }
}

9、纯注解格式

  • 名称：@Configuration、@ComponentScan

  • 类型：类注解

  • 位置：类定义上方

  • 作用：设置当前类为spring核心配置加载类

  • 范例：

    @Configuration
@ComponentScan("scanPackageName")
public class SpringConfigClassName{
}

  • 说明：

    • 核心配合类用于替换spring核心配置文件，此类可以设置空的，不设置变量与属性
    • bean扫描工作使用注解@ComponentScan替代

AnnotationConfigApplicationContext

  • 加载纯注解格式上下文对象，需要使用AnnotationConfigApplicationContext

    AnnotationConfigApplicationContext ctx = new AnnotationConfigApplicationContext(SpringConfig.class);

代码演示：

/*
*  @Configuration: 设置一个类为Spring的配置类
 * @ComponentScan: 设置需要扫描的包
* */
@Configuration
@ComponentScan("com.itheima")
@PropertySource("classpath:jdbc.properties")
public class SpringConfig {
}

测试类：

public class App {
    public static void main(String[] args) {
//        ApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
        
//加载纯注解格式上下文对象，需要使用AnnotationConfigApplicationContext
        ApplicationContext ctx = new AnnotationConfigApplicationContext(SpringConfig.class);
        MyClass mc = (MyClass) ctx.getBean("fb");
        mc.run();

    }
}

10、第三方bean配置与管理

  • 名称：@Import

  • 类型：类注解

  • 位置：类定义上方

  • 作用：导入第三方bean作为spring控制的资源

  • 范例：

    @Configuration
@Import(OtherClassName.class)
public class ClassName {
}

  • 说明：

    • @Import注解在同一个类上，仅允许添加一次，如果需要导入多个，使用数组的形式进行设定
    • 在被导入的类中可以继续使用@Import导入其他资源（了解）
    • @Bean所在的类可以使用导入的形式进入spring容器，无需声明为bean

SpringConfig类：

@Configuration
@ComponentScan("com.itheima")
//通过@Import导入第三方bean作为spring容器控制的资源
@Import(MyClassFactory.class)        //MyClassFactory上不需要再声明@Component
public class SpringConfig {
}

//刚才这里需要声明为bean,放到spring容器中
// 现在有了SpringConfig这个配置类,统一在那边配置即可
//@Component
public class MyClassFactory {

    @Bean("fb")
    public MyClass getMyClass(){
        MyClass mc = new MyClass();
        mc.setName("lisi");
        mc.setAge(18);
        return mc;
    }
}

JdbcTemplate是spring提供的一个模板类，它是对jdbc的封装,用于支持持久层的操作。

maven坐标：

1、入门案例

public class Jdbc02Test {

    //JdbcTemplate的入门案例
    @Test
    public void test() throws Exception {
        //1.导入jar包坐标
        //2.创建JdbcTemplate
        // 该对象的执行需要依赖数据源(连接池),该对象会自动从连接池中获取连接,并使用获取的连接
        JdbcTemplate template = new JdbcTemplate();
        //3.创建连接池对象,交个template使用
        DriverManagerDataSource dataSource = new DriverManagerDataSource();
        dataSource.setUsername("root");
        dataSource.setPassword("root");
        dataSource.setUrl("jdbc:mysql://localhost:3306/class120");
        dataSource.setDriverClassName("com.mysql.jdbc.Driver");
        template.setDataSource(dataSource);
        //4.编写sql语句
        String sql = "select * from account where id = ? ";
        //5.执行sql并返回结果集
        // 参数1: 要执行的sql语句
        // 参数2: 返回结果类型
        // 参数3: 可变参,
        //  sql语句中有几个? 此处就有几个参数,
        //  参数的个数,顺序,类型要和sql语句中? 的个数,顺序,类型保持一致
        Account account = template.queryForObject(sql,new BeanPropertyRowMapper<Account>(Account.class), 1);
        System.out.println(account);
    }

}

2、 JdbcTemplate的DML操作

# 1. DML操作
1. DML: data manipulation language 数据操纵语言
2. DML包含了增删改三种操作(insert,delete,update)

# 2. template对象的API
    public int update(final String sql,Object... args)
    1. 参数 sql: 要执行的sql语句
    2. 参数 args: sql执行时需要的参数
    3. 返回值 : 被影响的行数

代码：

public class Jdbc03Test {
    JdbcTemplate template;

    //在每个@Test之前执行一次
    @Before
    public void before() {
        template = new JdbcTemplate();
        //创建连接池对象,交个template使用
        DriverManagerDataSource dataSource = new DriverManagerDataSource();
        dataSource.setUsername("root");
        dataSource.setPassword("root");
        dataSource.setUrl("jdbc:mysql://localhost:3306/class120");
        dataSource.setDriverClassName("com.mysql.jdbc.Driver");
        template.setDataSource(dataSource);
    }

    /**
     * 添加:
     *      需求: 向数据库中添加一条记录 柳岩 999
     *  API:
     *      update(String sql,Object... arg)
     *          参数1: 要执行的sql语句
     *          参数2: 可变参
     */
    @Test
    public void insert() {
        String sql = "insert into account values(null,?,?) ";
        int i = template.update(sql, "柳岩", 999);
        if(i>0){
            System.out.println("添加成功...");
        }
    }

    /**
     * 修改:
     *      需求: 修改id为3的账户名为:大幂幂 金额:1999
     */
    @Test
    public void update() {
        String sql = "update account set name=?,money=? where id=? ";
        Object params[] = {"大幂幂",1999,3};
        int i = template.update(sql, params);
        if(i>0){
            System.out.println("修改成功...");
        }
    }

    /**
     * 删除:
     *      需求: 删除id为4的账户信息
     *
     */
    @Test
    public void delete() {
        String sql = "delete from account where id=? ";
        int i = template.update(sql, 3);
        if(i>0){
            System.out.println("删除成功...");
        }
    }

}

3、JdbcTemplate的DQL操作

# 1. DQL介绍
1. DQL: data query language 数据查询语言
2. DQL包含了查询操作: select

# 2. DQL结果集分类
    对DQL执行结果进行分类的话, 大致可以分为以下三类
1. 单行单列: 基本类型或String接收结果即可
        select name from 表名 where id = 1;
        select count(*) from 表名;
2. 单行多列: Map<key,value> key为字段名称,value为查询到的结果
        select * from 表名 where id = 1;
3. 多行多列: List<Map<key,value>>
        select * from 表名;
# 3. template对象的API
    对应不同的结果集类型,有以下的方法可以使用
1. 单行单列
    public <T> T queryForObject(String sql, Class<T> requiredType, Object... args)
        sql: 要执行的sql
        requiredType: 返回的类型的字节码对象
        args: sql所需要的参数
2. 单行多列    
    public Map<String, Object> queryForMap(String sql, Object... args)
        sql: 要执行的sql
        args: sql所需要的参数
        注意: 此方法只能返回一条结果!
3. 多行多列
    List<Map<String, Object>> queryForList(String sql, Object... args)
        sql: 要执行的sql
        args: sql所需要的参数
        //将每条记录放map中,再将每个map放入list中
4. 多行多列        
    public <T> List<T> query(String sql, RowMapper<T> rowMapper,Object... args) 
           sql: 要执行的sql
           rowMapper: 行映射器
                  我们可以通过BeanPropertyRowMapper并指定对应的返回值类型
                  将结果集的每一行记录映射到一个对应类型的javaBean中
        args: sql所需要的参数
        //现将每一条记录封装到bean中,将多个bean使用list封装
5. sql说明          
        sql中可以使用?占位符

 JdbcTemplate操作 简洁版
 1. DML
         insert / update / delete
 2. DQL
     1). 结果集为单行单列  :
         String = queryForObject (sql,String.class,参数)
     2). 结果集为单行多列 :
          Map =    queryForMap(sql,参数)
          Account =    queryForObject(sql, new BeanPropertyRowMapper<>(Account.class), 参数)
     3). 结果集为多行多列:
          List<Map>  =  queryForList(sql,参数)
           List<Account> =    query(sql, new BeanPropertyRowMapper<>(Account.class), 参数)

代码：

/*
    JdbcTemplate
    1. DML
            insert / update / delete
    2. DQL
        1). 结果集为单行单列  :
            String = queryForObject (sql,String.class,参数)
        2). 结果集为单行多列 :
             Map =    queryForMap(sql,参数)
             Account =    queryForObject(sql, new BeanPropertyRowMapper<>(Account.class), 参数)
        3). 结果集为多行多列:
             List<Map>  =  queryForList(sql,参数)
              List<Account> =    query(sql, new BeanPropertyRowMapper<>(Account.class), 参数)
 */
public class JdbcTemplateDemo2 {
    private static DruidDataSource ds;
    static{
        ds = new DruidDataSource();
        ds.setUrl("jdbc:mysql://localhost:3306/class134");
        ds.setDriverClassName("com.mysql.jdbc.Driver");
        ds.setUsername("root");
        ds.setPassword("root");
    }
    @Test
    public void queryForObject(){
        String sql = "select name from account where id = ?";
        JdbcTemplate template = new JdbcTemplate(ds);
        /*
            1. String sql
            2. Class 返回值类型
            3. Object... args 执行sql需要的参数
         */
        String name = template.queryForObject(sql, String.class, 1);
        System.out.println(name); // jack
    }

    @Test
    public void queryForObject2(){
        String sql = "select count(*) from account";
        JdbcTemplate template = new JdbcTemplate(ds);
        int count = template.queryForObject(sql, int.class);
        System.out.println(count);// 3
    }

    @Test
    public void queryForMap(){
        String sql = "select * from account where id = ?";
        JdbcTemplate template = new JdbcTemplate(ds);
        Map<String, Object> map = template.queryForMap(sql, 2);
        //{id=2, name=rose, money=21000.0}
        System.out.println(map);
    }

    @Test
    public void queryForObject3(){
        String sql = "select * from account where id = ?";
        JdbcTemplate template = new JdbcTemplate(ds);
        /*
            1. String sql
            2. RowMapper 行映射器
                实现ORM : 将结果集的行映射到指定javaBean
            3. Object... args 执行sql需要的参数
         */
        Account account = template.queryForObject(sql, new BeanPropertyRowMapper<>(Account.class), 1);
        //Account{id=1, name='jack', money=1000.0}
        System.out.println(account);
    }

    @Test
    public void queryForList(){
        String sql = "select * from account";
        JdbcTemplate template = new JdbcTemplate(ds);
        List<Map<String, Object>> list = template.queryForList(sql);
        //{id=1, name=jack, money=1000.0}
        //{id=2, name=rose, money=21000.0}
        //{id=3, name=tom, money=5000.0}
        for (Map<String, Object> map : list) {
            System.out.println(map);
        }
    }
    @Test
    public void query(){
        String sql = "select * from account";
        JdbcTemplate template = new JdbcTemplate(ds);
        List<Account> list = template.query(sql, new BeanPropertyRowMapper<>(Account.class));
        //Account{id=1, name='jack', money=1000.0}
        //Account{id=2, name='rose', money=21000.0}
        //Account{id=3, name='tom', money=5000.0}
        for (Account account : list) {
            System.out.println(account);
        }
    }
}

3、纯注解方式

1. 第三方jar包提供的类也用注解配置
2. 那么核心配置文件就不需要了,用配置类代替

SpringConfig类

//spring注解配置类
@Configuration
@ComponentScan("com.itheima") // 包扫描
@PropertySource("classpath:jdbc.properties") // 加载properties配置
public class SpringConfig {

    @Value("${jdbc.url}")
    private String url;
    @Value("${jdbc.driver}")
    private String driverClassName;
    @Value("${jdbc.username}")
    private String username;
    @Value("${jdbc.password}")
    private String password;

    // 此DataSource会被放到IOC容器中
    @Bean("ds")
    public DataSource getDs(){
        DruidDataSource ds = new DruidDataSource();
//        ds.setUrl("${jdbc.url}"); // 这么写是不行的
        ds.setUrl(url);
        ds.setDriverClassName(driverClassName);
        ds.setUsername(username);
        ds.setPassword(password);
        return ds;
    }

    //参数会去IOC容器中获取
    @Bean("template")
    public JdbcTemplate getTemplate(DataSource ds){
//        DataSource ds = getDs();
        JdbcTemplate template = new JdbcTemplate();
        template.setDataSource(ds);
        return template;
    }
}
